# MVP 요구사항

## 목표

My PT AI는 개인 사용자가 식단, 운동, 하루 컨디션을 기록하고, 최근 기록을 바탕으로 AI 코칭을 받을 수 있게 하는 Spring Boot 웹앱입니다.

첫 버전은 의도적으로 작게 만듭니다. 로그인, 배포, 결제, 모바일 PWA, 고급 통계보다 먼저 핵심 사용 흐름이 실제로 동작하는지 검증합니다.

## 핵심 사용자 흐름

1. 사용자가 웹앱에 접속합니다.
2. 사용자가 기록할 날짜를 선택합니다.
3. 사용자가 해당 날짜의 식단을 기록합니다.
4. 사용자가 해당 날짜의 운동을 기록합니다.
5. 사용자가 수면, 컨디션, 메모 같은 하루 상태를 기록합니다.
6. 사용자가 AI에게 내일 식단과 운동 추천을 요청합니다.
7. 백엔드는 필요한 프로필과 최근 기록만 OpenAI API로 전달합니다.
8. 사용자는 AI 답변을 확인하고, 이전 코칭 기록도 다시 볼 수 있습니다.

## MVP 범위

### 사용자 프로필

- 로컬 사용자 프로필 1개를 생성하고 수정할 수 있습니다.
- 이름, 목표, 키, 체중, 활동량, 선호 식단, 제한사항을 저장합니다.
- MVP에서는 회원가입과 로그인은 포함하지 않습니다.

### 식단 기록

- 날짜별 식단 기록을 생성할 수 있습니다.
- 날짜별 식단 기록을 조회할 수 있습니다.
- 식단 기록을 수정할 수 있습니다.
- 식단 기록을 삭제할 수 있습니다.
- 식사 구분, 메뉴명, 칼로리, 단백질, 탄수화물, 지방, 메모를 저장합니다.

### 운동 기록

- 날짜별 운동 기록을 생성할 수 있습니다.
- 날짜별 운동 기록을 조회할 수 있습니다.
- 운동 기록을 수정할 수 있습니다.
- 운동 기록을 삭제할 수 있습니다.
- 운동 종류, 운동명, 운동 시간, 강도, 메모를 저장합니다.

### 하루 컨디션

- 날짜별 컨디션 기록을 생성하거나 수정할 수 있습니다.
- 날짜마다 컨디션 기록은 1개만 존재합니다.
- 수면 시간, 컨디션, 당일 체중, 메모를 저장합니다.

### AI 코칭

- 사용자는 자유롭게 질문을 입력할 수 있습니다.
- 백엔드는 사용자 프로필, 선택 날짜, 최근 기록을 바탕으로 프롬프트를 구성합니다.
- 백엔드는 OpenAI API를 호출합니다.
- 질문, 답변, 사용 모델, 요청 상태를 저장합니다.

## MVP 제외 범위

- 로그인과 다중 사용자 인증
- 결제
- 관리자 페이지
- 앱스토어 배포
- 푸시 알림
- 사진 기반 칼로리 인식
- 웨어러블 기기 연동
- 전체 영양성분 데이터베이스

## 완료 기준

- `./gradlew bootRun`으로 앱이 실행됩니다.
- H2 로컬 프로필은 별도 설치 없이 동작합니다.
- JPA 스키마는 Hibernate 자동 생성이 아니라 Flyway로 관리합니다.
- Entity를 폼 객체로 직접 사용하지 않습니다.
- 검증 실패 시 같은 Thymeleaf 폼으로 돌아가고 필드 오류가 표시됩니다.
- 트랜잭션 경계는 Service 계층에서 관리합니다.
- OpenAI API 키는 환경변수에서만 읽습니다.
- 각 핵심 도메인은 최소한 Service 동작 테스트를 가집니다.
